# Constitution準拠コミット

**`../docs/development.md` に記載されているすべてのルールを厳守してコミットを実行してください。**

## 実行モードの判定

`ARGUMENTS` に含めるファイル（またはコミット対象）が指定されているかで動作を分ける:

- **指定あり（自走モード）**: ファイル選択もコミットメッセージの承認も**ユーザーに確認せず**実行する。linear-implement スキル等から invoke される場合はこちら
- **指定なし（対話モード）**: 従来どおりユーザーにファイル選択を依頼し、メッセージの承認を得てからコミットする

## 実行手順

### 1. 現在の変更を確認

!`git status`

### 2. コミット対象ファイルの決定

- **自走モード**: `ARGUMENTS` で渡されたファイルをそのまま対象とする。ユーザーへの選択依頼はしない
- **対話モード**: 上記の変更ファイルから、どのファイルを add/commit するか選択してもらう
  - 複数ファイル選択可能 / すべて選択も可能 / ファイルパスを正確に確認

### 3. 対象ファイルを add

決定したファイルを `git add` してください。

### 4. ステージング内容を確認

!`git diff --staged`

### 5. Commitメッセージの生成とコミット

[development.md](../docs/development.md) のGit規約を参照してコミットメッセージを生成してください。

- **自走モード**: メッセージ案を提示したうえで、承認を待たずそのままコミットを実行する
- **対話モード**: メッセージ案を提示し、ユーザーの承認を得られたらコミットを実行する
